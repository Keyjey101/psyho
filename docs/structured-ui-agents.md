# PsyHo — План улучшений v2: UI/UX + Поведение агентов

## Context

Предыдущий план (v1) выполнен. Сейчас: ограниченные сессии (20 обменов), SessionPhase, компактный ActionPanel, фоновая память, lazy-load истории. Нужен следующий уровень качества.

Исследованы: все chat-компоненты, Onboarding, Profile, MoodTracker, BreathingExercise, все 6 промптов агентов, orchestrator.py, memory_service.py, /continue и /insights эндпоинты.

## Блок 1: UI/UX — Завершение терапевтического цикла

**Проблема:** `MoodTracker.tsx` и `MoodPage.tsx` существуют, но никогда не вызываются из основного потока.

**Решение:**

- `ActionPanel.tsx`: после закрытия BreathingExercise (onComplete колбэк) — показывать `MoodTracker` поверх
- `Sidebar.tsx`: добавить ссылку на `/mood` рядом с «Профиль»

**Файлы:** `Chat.tsx`, `ActionPanel.tsx`, `Sidebar.tsx`, `MoodTracker.tsx`

## Блок 2: Поведение агентов — Структурные улучшения

### 2.1 Фазы сессии — структурное влияние (не косметика)

**Проблема:** Текущие фазы — это 15 слов в `user_content`. INTAKE всё равно запускает агентов. CLOSE не проверяет понимание.

**Решение:**

**INTAKE (обмены 1–3):** агенты НЕ запускаются. Оркестратор работает один.
```python
# orchestrator.py: process()
if phase == SessionPhase.INTAKE:
    selected = []  # No agents in intake
```

В synthesis-промпт для INTAKE добавить конкретную инструкцию:
```
ЗАДАЧА INTAKE: Задай 1-2 уточняющих вопроса чтобы понять:
1. Что именно беспокоит (конкретная ситуация, не общая тема)
2. Чего человек хочет от этой сессии (понять / решить / просто выговориться)
НЕ давай советов. НЕ анализируй. Слушай и уточняй.
```

**CLOSE (последние 2 обмена):** synthesis-промпт получает структурную инструкцию:
```
ЗАДАЧА CLOSE: Это последний обмен. Сделай:
1. Краткое резюме: что прояснилось, что изменилось в понимании
2. 1 конкретная домашняя задача / практика до следующей сессии
3. Тёплое завершение, приглашение вернуться
```

**INTEGRATION (обмены 15–18):** synthesis-промпт:
```
ЗАДАЧА INTEGRATION: Начинай закреплять. Предложи 1 конкретное упражнение
(если соматика — укажи технику из 4-7-8 или 5-4-3-2-1).
```

**Файлы:** `orchestrator.py`, `orchestrator.txt`

### 2.2 Исправить топик-маппинг агентов

**Текущие пробелы:**
- `trauma` → только `[somatic, ifs]`, нет Narrative (нарративная терапия — основной метод для травмы)
- `dreams` → только `[jungian]`, нет Somatic (сомнические переживания имеют телесный компонент)
- `grief` → `[narrative, somatic]`, нет ACT (принятие потери — ключевой ACT-процесс)
- Отсутствуют топики: `shame` (стыд), `perfectionism` (перфекционизм отдельно от `self_esteem`), `loneliness_chronic` (хроническое одиночество)

**Решение:**
```python
TOPIC_AGENT_MAP = {
    ...существующие...
    "trauma": ["somatic", "narrative"],    # было [somatic, ifs]
    "dreams": ["jungian", "somatic"],      # было [jungian]
    "grief": ["narrative", "act"],         # было [narrative, somatic]
    # новые топики:
    "shame": ["ifs", "narrative"],
    "perfectionism": ["cbt", "ifs"],
}
```

Также обновить classification prompt — добавить новые топики в список.

**Файлы:** `orchestrator.py`

### 2.3 Единая личность Ники — убрать агентские метки из истории

**Проблема:** История передаётся оркестратору с пометками `[через КПТ, Соматика]`. Это нарушает иллюзию единой Ники и поощряет LLM цитировать агентов.

**Решение:**
```python
# orchestrator.py:_synthesize() — убрать agents_note из истории
for m in history[-10:]:
    role = "Пользователь" if m["role"] == "user" else "Ника"
    history_text += f"{role}: {m['content']}\n"
    # agents_note убран
```

**Файлы:** `orchestrator.py`

### 2.4 Консолидация промптов агентов — общая преамбула

**Проблема:** ~50% каждого промпта (~400 токенов) — идентичный boilerplate.

**Решение:**

Добавить в `base.py` общую преамбулу, которая вставляется ПЕРЕД `self.system_prompt` каждого агента:
```python
AGENT_PREAMBLE = """Ты — эксперт-аналитик. Твой анализ читает главный терапевт Ника — она синтезирует все перспективы в единый ответ пользователю. Ты НЕ общаешься с пользователем напрямую.

Пиши аналитически, конкретно. Без вступлений «Конечно!» или «Я помогу».
Язык: всегда русский.
Объём: 300-400 слов.
Выполни цепочку рассуждений ПЕРЕД тем как писать анализ."""
```

Из каждого `.txt` промпта удалить повторяющиеся части — оставить только специфику модальности. Экономия: ~400 токенов × 2 агента = ~800 токенов/запрос.

**Файлы:** `base.py`, все 6 `prompts/*.txt`

### 2.5 Улучшение /continue — структурированные инсайты

**Проблема:** `/continue` берёт только ответы Ники (не запросы пользователя), первые 500 символов каждого — и просит LLM претендовать, что Ника «помнит» это. Это противоречит orchestrator.txt строка 70.

**Решение:**

1. Брать ОБА: последние 10 пар (user + assistant)
2. Структурированный промпт с JSON-выходом:
```python
CONTINUE_PROMPT = """Из этой сессии извлеки выжимку для следующей встречи.

Верни JSON:
{
  "main_theme": "одна фраза — главная тема",
  "user_request": "что человек хотел получить от сессии",
  "key_insights": ["инсайт 1", "инсайт 2"],
  "homework": "конкретная практика если была",
  "continue_from": "с чего начать следующий разговор — 1 предложение"
}"""
```

3. В greeting нового сессии — использовать `continue_from` как отправную точку, не «я помню, мы говорили о...»

**Файлы:** `routers/sessions.py`, `routers/messages.py` (continuation greeting)

### 2.6 Улучшение памяти — временное закрепление

**Проблема:** Long-term memory не различает «раньше» и «сейчас». Экстракция теряет временной контекст.

**Решение — изменить формат памяти:**
```
MEMORY_EXTRACT_PROMPT добавить инструкцию:
"Помечай временной контекст: [тогда] для прошлого, [сейчас] для текущего.
Пример: '[тогда] боялся публичных выступлений → [сейчас] чувствует прогресс'.
Фиксируй изменения, а не только факты."
```

Также добавить поле `memory_updated_at` в `UserProfile` — чтобы знать, когда обновлялась.

**Файлы:** `services/memory_service.py`, `models/models.py`

### 2.7 Качество промптов агентов — точечные правки

Конкретные улучшения без полного переписывания:

**`ifs.txt`:** Убрать строку с когнитивным слиянием (ACT-язык, не IFS). Заменить на:
```
"Какая часть сейчас говорит? (Менеджер, Изгнанник, Пожарный)"
```

**`jungian.txt`:** Добавить критерий применимости:
```
"Если человек описывает конкретную бытовую ситуацию без символики, сновидений или проекций — 
скажи об этом в ответе. Не натягивай юнгианские концепции."
```

**`narrative.txt`:** Добавить обработку отсутствия исключений:
```
"Если человек не может вспомнить ни одного момента, когда было иначе — 
это важная информация. Предложи Нике отметить это и не форсировать поиск."
```

**`somatic.txt`:** Добавить диссоциацию в ограничения:
```
"Если человек описывает онемение, отстранённость, 'всё как в тумане' — 
приоритет: заземление через зрение и твёрдые поверхности, не через прикосновения."
```

**Файлы:** `ifs.txt`, `jungian.txt`, `narrative.txt`, `somatic.txt`

## Верификация

1. **Структурные фазы:** отправить 3 сообщения → агенты не запускаются (agents_used = ["orchestrator"]) → WS event должен содержать `agents: ["orchestrator"]` в первых 3 обменах
2. **SessionEndCard:** довести сессию до лимита → появляется карточка с mood + continue/end → выбор настроения сохраняется в БД
3. **Тултипы:** hover на бейдж «КПТ» → появляется tooltip с объяснением
4. **История без меток:** в logs синтез-промпта не должно быть `[через КПТ]` в строках истории
5. **Инсайты /continue:** POST `/sessions/{id}/continue` → response содержит `main_theme`, `key_insights` как JSON-поля (не freeform)
6. **Промпты:** отправить сообщение про травму → убедиться что narrative агент тоже вызывается (агент выбирается из обновлённого маппинга)
